🚀 راهنمای اجرای Vercel XHTTP

راهنمای اختصاصی برای کاربرانی که x-ui و xray را نصب کرده‌اند

✅ آماده اجرا
مرور کلی پیش‌نیازها تنظیم x-ui دیپلوی روی Vercel کانفیگ کلاینت عیب‌یابی

📋 مرور کلی

از آنجایی که شما قبلاً x-ui و xray را روی سرورهای خود نصب کرده‌اید، بسیاری از مراحل اولیه (نصب Xray، گرفتن گواهی SSL و...) را انجام داده‌اید. این راهنما فقط مراحل باقی‌مانده برای اجرای پروژه Vercel XHTTP را پوشش می‌دهد.

ℹ️

معماری سیستم

کلاینت → Vercel Edge (پنهان‌ساز IP) → سرور Xray شما → اینترنت

✅

وضعیت فعلی شما

شما VPS با Ubuntu 24 دارید و x-ui با هسته xray نصب شده است. این یعنی:
• سرور Xray آماده است
• احتمالا گواهی SSL دارید
• فقط باید کانفیگ XHTTP اضافه و Vercel را تنظیم کنید

🔧 پیش‌نیازها

۱

اکانت Vercel

اگر اکانت ندارید، از vercel.com رایگان ثبت‌نام کنید.

۲

دسترسی SSH به سرور Ubuntu 24

شما باید بتوانید با SSH به سرور خارجی خود وصل شوید:

ssh root@YOUR_VPS_IP
۳

دامنه با DNS به سرور

یک دامنه (پولی یا رایگان مثل DuckDNS) نیاز دارید که A record آن به IP سرور خارجی شما اشاره کند.

⚠️

نکته مهم

اگر از Cloudflare استفاده می‌کنید، Proxy status را روی DNS only (خاکستری) بگذارید، نه Proxied.

⚙️ تنظیم x-ui برای XHTTP

۱

ورود به پنل x-ui

به پنل x-ui سرور خارجی خود وارد شوید.

۲

ایجاد اینباند جدید

به بخش Inbounds بروید و یک اینباند جدید با تنظیمات زیر ایجاد کنید:

تنظیم مقدار
Protocol VLESS
Port 2096 (یا پورت دلخواه)
Network XHTTP
Security TLS
Path /yourpath (مسیر دلخواه)
Host دامنه شما (مثلاً xray.yourdomain.com)
۳

تنظیم گواهی SSL

در بخش TLS settings:

  • Certificate File: مسیر cert.pem
  • Key File: مسیر key.pem
  • اگر cert ندارید، با acme.sh بگیرید:
curl https://get.acme.sh | sh -s email=your@email.com source ~/.bashrc ~/.acme.sh/acme.sh --issue -d xray.yourdomain.com --standalone -k ec-256 ~/.acme.sh/acme.sh --install-cert -d xray.yourdomain.com --ecc \ --fullchain-file /etc/xray/cert.pem \ --key-file /etc/xray/key.pem
۴

باز کردن پورت در فایروال

ufw allow 2096/tcp ufw reload
۵

تست کانفیگ

curl -vk https://127.0.0.1:2096/yourpath

باید HTTP/2 404 دریافت کنید (این یعنی Xray کار می‌کند).

🌐 دیپلوی روی Vercel (گام به گام)

ℹ️

روش دیپلوی

ما از روش Vercel CLI روی سرور Ubuntu 24 استفاده می‌کنیم.

۱

نصب Node.js و npm روی سرور Ubuntu 24

ابتدا با SSH به سرور خارجی خود وصل شوید:

ssh root@YOUR_VPS_IP

ابتدا چک کنید Node.js نصب است یا نه:

node --version npm --version

اگر نصب نبود، این دستورات را بزنید:

sudo apt update sudo apt install -y nodejs npm

بعد از نصب، دوباره نسخه را چک کنید:

node --version npm --version
۲

نصب Vercel CLI روی سرور

این دستور را روی سرور Ubuntu 24 بزنید:

sudo npm install -g vercel

تست نصب:

vercel --version

باید نسخه‌ای مثل 32.x.x یا مشابه ببینید.

۳

لاگین به Vercel

vercel login

مراحل لاگین:

  1. با دکمه‌های جهت‌نما (↑ ↓) گزینه Continue with Email را انتخاب کنید و Enter بزنید
  2. ایمیل خود را وارد کنید و Enter بزنید
  3. به ایمیل خود بروید و لینک تأیید (Verify) را کلیک کنید
  4. بعد از کلیک، به ترمینال برگردید - باید پیام "Logged in as..." ببینید
✅

موفقیت

اگر پیام "Logged in" دیدید، لاگین موفق بوده است.

۴

کپی کردن پروژه روی سرور

ابتدا پروژه را از ویندوز به سرور منتقل کنید. چند روش وجود دارد:

⚠️

روش ۱: استفاده از SCP (از ویندوز به سرور)

در PowerShell یا Git Bash روی ویندوز این دستور را بزنید:

scp -r c:\Users\-Mahdi\Downloads\vercel\Vercel-XHTTP-main root@YOUR_VPS_IP:/root/
⚠️

روش ۲: استفاده از Git روی سرور

روی سرور این دستورات را بزنید:

cd /root git clone https://github.com/YOUR-USERNAME/vercel-xhttp-relay.git cd vercel-xhttp-relay

رفتن به پوشه پروژه روی سرور:

cd /root/Vercel-XHTTP-main

تست: با دستور ls باید فایل‌های زیر را ببینید:

  • api/
  • package.json
  • vercel.json
  • README.md
۵

دیپلوی اولیه (Preview)

در پوشه پروژه، دستور زیر را بزنید:

vercel

سؤالاتی که پرسیده می‌شود:

سؤال پاسخ توضیح
Set up and deploy? Y می‌خواهید پروژه را ستاپ کنید
Which scope? ایمیل شما اکانت خودتان را انتخاب کنید
Link to existing project? N پروژه جدید می‌سازیم
What's your project's name? اسم دلخواه مثلاً: my-xhttp-relay
In which directory? ./ (Enter) همان پوشه فعلی
Want to modify settings? N تنظیمات پیش‌فرض خوب است

بعد از اتمام، URL مثل این دریافت خواهید کرد:

https://my-xhttp-relay-abc123.vercel.app

این URL Preview است - هنوز برای استفاده نهایی آماده نیست.

۶

تنظیم Environment Variable (TARGET_DOMAIN)

⚠️

بسیار مهم - حیاتی

این متغیر آدرس سرور Xray شما را به Vercel می‌دهد. اگر اشتباه باشد، کل سیستم کار نمی‌کند.

فرمت صحیح:

https://YOUR-DOMAIN.COM:PORT

مثال:

https://xray.mydomain.com:2096

حالا دستور زیر را بزنید:

vercel env add TARGET_DOMAIN

مراحل تنظیم:

  1. مقدار (value) را وارد کنید: https://xray.yourdomain.com:2096 و Enter
  2. Environments: با Space تیک کنید:
    • ✅ Production (با Space تیک بزنید)
    • ✅ Preview (با Space تیک بزنید)
    • ❌ Development (تیک نزنید!)
  3. بعد از تیک زدن، Enter بزنید
  4. Git branch: خالی بگذارید و Enter بزنید
  5. Make sensitive: n بزنید (Enter)
✅

تأیید

باید پیام "Added environment variable TARGET_DOMAIN" ببینید.

برای مشاهده متغیرهای ست شده:

vercel env ls
۷

دیپلوی نهایی (Production)

حالا با متغیر محیطی ست شده، دیپلوی Production انجام دهید:

vercel --prod

این دستور ممکن است 30-60 ثانیه طول بکشد. منتظر بمانید تا تمام شود.

بعد از اتمام:

  • URL Production را کپی کنید (مثل: https://my-xhttp-relay.vercel.app)
  • این URL را ذخیره کنید - برای کانفیگ کلاینت لازم است
✅

موفقیت

پروژه شما روی Vercel دیپلوی شد! URL را یادداشت کنید.

۸

غیرفعال کردن Deployment Protection

⚠️

ضروری - بدون این کار نمی‌کند

اگر Deployment Protection روشن باشد، Vercel قبل از فوروارد کردن ترافیک، صفحه لاگین نشان می‌دهد و relay کار نمی‌کند.

مراحل در داشبورد Vercel:

  1. به vercel.com/dashboard بروید
  2. پروژه خود را پیدا کنید و کلیک کنید
  3. در منوی بالا، روی Settings کلیک کنید
  4. از منوی سمت چپ، Deployment Protection را پیدا کنید
  5. گزینه Vercel Authentication را پیدا کنید
  6. آن را روی Disabled بگذارید (کلیک کنید → Disabled را انتخاب کنید)
  7. روی دکمه Save کلیک کنید
ℹ️

تغییرات دیگر

همچنین می‌توانید "Password Protection" و "IP Allowlist" را هم Disabled کنید تا مطمئن شوید هیچ مانعی وجود ندارد.

۹

تست relay

حالا تست کنید که آیا relay کار می‌کند یا نه:

# جای your-project.vercel.app و yourpath را با مقادیر واقعی جایگزین کنید curl -I https://your-project.vercel.app/yourpath

معنی خروجی‌ها:

خروجی وضعیت معنی
HTTP/2 404 ✅ عالی Relay کار می‌کند (404 طبیعی است چون UUID نفرستادید)
HTTP/2 401 + HTML login ❌ مشکل Deployment Protection هنوز روشن است
HTTP/2 500 ❌ مشکل TARGET_DOMAIN ست نشده یا اشتباه است
HTTP/2 502 ❌ مشکل سرور Xray در دسترس نیست یا پورت بسته است
HTTP/2 504 ❌ مشکل تایم‌اوت - سرور خیلی کند است یا پاسخ نمی‌دهد
✅

اگر HTTP/2 404 دیدید

تبریک! relay شما آماده است. به بخش کانفیگ کلاینت بروید.

۱

مقادیر کانفیگ

تنظیم مقدار

Port 443
SNI vercel.com
Type xhttp
Path /yourpath
Host your-project.vercel.app
Mode auto
TLS on
ALPN h2

۲

لینک VLESS

vless://YOUR-UUID@vercel.com:443?encryption=none&security=tls&sni=vercel.com&alpn=h2&fp=chrome&type=xhttp&path=%2Fyourpath&host=your-project.vercel.app&mode=auto#Vercel-Relay

مقادیر YOUR-UUID، yourpath و your-project.vercel.app را با مقادیر واقعی جایگزین کنید.

۴

نکات مهم تنظیم

⚠️

حیاتی

  • Core: حتماً Xray-core (نه V2Ray-core)
  • Mux: خاموش (OFF)
  • Routing: Bypass LAN/Iran را روشن کنید
  • Allow Insecure: OFF
۵

تست اتصال

بعد از اتصال، به ifconfig.me بروید.

باید IP سرور VPS خارجی خود را ببینید، نه IP ایران.

🔍 عیب‌یابی

۱

502 Bad Gateway: Tunnel Failed

  • TARGET_DOMAIN را چک کنید (دقیقاً https://...:port)
  • Xray روی سرور بالا باشد: systemctl status xray
  • پورت در فایروال باز باشد: ufw status
۲

500 Misconfigured: TARGET_DOMAIN is not set

vercel env ls vercel --prod
۳

401 Unauthorized با HTML login

Vercel Authentication روشن است. در داشبورد → Settings → Deployment Protection → Disabled

۴

کلاینت وصل می‌شود ولی ترافیک رد نمی‌شود

  • Mux را خاموش کنید
  • Core را روی Xray-core بگذارید
  • Routing → Bypass Iran را فعال کنید
۵

TLS handshake error در کلاینت

  • SNI را از vercel.com به your-project.vercel.app عوض کنید
  • ALPN را فقط h2 بگذارید
۶

مشاهده لاگ Vercel

vercel logs --follow

یا در داشبورد → پروژه → Logs

❓ سوالات متداول

آیا می‌توانم دامنه شخصی به Vercel وصل کنم؟

بله. در داشبورد Vercel → Settings → Domains → Add دامنه خود را اضافه کنید. سپس در کلاینت host و address را همان دامنه بگذارید.

چند کاربر می‌تواند همزمان وصل شود؟h3>

محدودیت سختی نیست. در x-ui برای هر کاربر یک UUID جداگانه بسازید.

آیا می‌توانم پورت را عوض کنم؟

بله. پورت دلخواه را در x-ui بگذارید، در ufw باز کنید، و در TARGET_DOMAIN در Vercel همان پورت را بگذارید.

Certificate تمدید خودکار می‌شود؟

بله. acme.sh هر 60 روز خودکار renew می‌کند.
